# Step 2 — Fix the sitemap.xml

The current sitemap has three problems:
1. Every article has `<changefreq>never</changefreq>` — Google reads this as "abandoned content"
2. Several articles have `<lastmod>` dates from 2025 that look like typos (the site went live in March 2026)
3. All non-homepage URLs have the same priority of 0.8 — no differentiation

Rewrite `sitemap.xml` (or the sitemap-generation logic, if dynamic) with the following policy:

| URL type | changefreq | priority |
|---|---|---|
| Homepage `/` | `weekly` | `1.0` |
| News index `/news` | `weekly` | `0.9` |
| Individual news posts `/news/*` | `weekly` | `0.9` |
| Pillar / hub pages (`/cost-of-living`, `/schools`, `/franklin-vs-nashville`, `/neighborhoods`) | `monthly` | `1.0` |
| Articles `/articles/*` (those NOT being 301'd — see Step 3) | `monthly` | `0.8` |
| Static pages (`/about` once created) | `yearly` | `0.5` |

For `lastmod` dates:
- If the article has a real publish/update date in its data, use that
- If the existing sitemap has a date like `2025-03-22` that's actually a typo for `2026-03-22`, correct it to the real year
- For pages without any reliable date, use today's date
- Use ISO 8601 format (`YYYY-MM-DD`)

Also: remove from the sitemap any URLs that will be 301'd in Step 3 (the `/articles/cost-of-living-franklin-tn`, `/articles/williamson-county-schools-guide`, `/articles/franklin-tn-vs-nashville`, `/articles/best-neighborhoods-in-franklin-tn` — these become redirects, not destinations).

Confirm `robots.txt` references the sitemap with a line like:
`Sitemap: https://franklintnguide.com/sitemap.xml`

If `robots.txt` doesn't exist, create it with that line plus `User-agent: *` / `Allow: /`.

Show me the diff before applying. Then apply and confirm the new sitemap renders correctly at https://franklintnguide.com/sitemap.xml.

# Step 6 — Add Article / NewsArticle schema to article and news pages

For every page under `/articles/*`, inject Article JSON-LD:
```json
{

  "@type": "Article",
  "headline": "{article.title}",
  "description": "{article.description}",
  "datePublished": "{article.publishDate}",
  "dateModified": "{article.updatedDate || article.publishDate}",
  "author": {
    "@type": "Person",

    "@type": "Organization",

      "@type": "ImageObject",
      "url": "https://franklintnguide.com/[actual-logo-path].png"
    }
  },
  "mainEntityOfPage": {
    "@type": "WebPage",
    "@id": "https://franklintnguide.com/articles/{article.slug}"
  },
  "image": "{article.heroImage || 'https://franklintnguide.com/og-default.jpg'}",
  "articleSection": "{article.category || 'Relocation Guide'}"
}
```

For every page under `/news/*`, use `NewsArticle` instead of `Article` (same fields, just the @type changes).

Rules:

- For `author.name`, use "Franklin TN Guide Editorial Team" as the default. If the article data has a real `author` field, use that instead.
- For `image`, prefer a real hero image. If none exists, point to a default Open Graph image at `/og-default.jpg` (note this in your report so I can create one).
